Custom Raspberry Pi Image

I recently became interested in deploying a project using a Raspberry Pi. My aim was to produce a project artifact that could be easily and reliably deployed to many Raspberry Pis.

Initial research led me to open source project that Raspbian is built from (Github . I also found some other projects that did not use Raspbian (such as gokrazy but I wanted to stick with something I had already tried and tested.

The official pi-gen git repo allowed me to build a whole fresh Raspbian image from scratch, using the STAGEs I was able to customise it and add my own software.

However, it wasn't long before I realised that this wasn't really what I wanted. The build proces took a long time (30 minutes) and because each build was pulling in packages from the central repository, the build changed over time even when I didn't want it to.

I soon realised that I actually wanted to base my build on the proper, versioned releases of raspbian (https://www.raspberrypi.org/downloads/raspbian/) but with some final some customisation on top.

The pi-gen scripts proved to be a very useful guide on how to achieve this. The gist is we mount the raspberry pi image, modify it, then unmount it. Simple.

Here's an example of what that might look like:

``` ROOTFS_DIR=rpi_mnt RPI_IMAGE=raspbian_lite_latest wget https://downloads.raspberrypi.org/raspbian_lite_latest

mkdir -p $ROOTFS_DIR LOOP_DEVICE=$(losetup -f -P --show ${RPI_IMAGE})p2
